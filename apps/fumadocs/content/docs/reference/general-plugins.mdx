---
title: General plugins
description: Build reusable send behavior without owning an adapter.
icon: Blocks
---

General plugins add behavior around existing adapters.

```ts
import type { EmailPlugin } from "email-sdk";

export function requireMetadataPlugin(key: string): EmailPlugin {
  return {
    id: `require-metadata:${key}`,
    middleware: [
      {
        beforeSend(event) {
          if (!event.options?.metadata?.[key]) {
            throw new Error(`Missing email metadata: ${key}`);
          }
        },
      },
    ],
  };
}
```

Good general plugin jobs include:

- Add default headers, tags, reply-to addresses, or metadata.
- Enforce policy before provider calls.
- Record test events.
- Emit logs, metrics, or traces with redacted message data.
- Add a small helper to the returned client.

Keep provider credentials inside adapter plugins or adapter options. General plugins should not hide provider-specific state that changes how a provider maps fields.
