Sending providers remain available from the package root subpaths.

| Provider                | Import                 | Capability     |
| ----------------------- | ---------------------- | -------------- |
| Resend                  | `email-sdk/resend`     | Send + receive |
| Postmark                | `email-sdk/postmark`   | Send + receive |
| SendGrid                | `email-sdk/sendgrid`   | Send + receive |
| Mailgun                 | `email-sdk/mailgun`    | Send + receive |
| SMTP                    | `email-sdk/smtp`       | Send only      |
| MailerSend              | `email-sdk/mailersend` | Send only      |
| Brevo                   | `email-sdk/brevo`      | Send only      |
| Mailchimp Transactional | `email-sdk/mailchimp`  | Send only      |
| SparkPost               | `email-sdk/sparkpost`  | Send only      |
| Loops                   | `email-sdk/loops`      | Send only      |
| Plunk                   | `email-sdk/plunk`      | Send only      |
| Mailtrap                | `email-sdk/mailtrap`   | Send only      |
| Scaleway                | `email-sdk/scaleway`   | Send only      |
| ZeptoMail               | `email-sdk/zeptomail`  | Send only      |
| MailPace                | `email-sdk/mailpace`   | Send only      |

Provider-specific pages remain available under `/docs/adapters/*`. The Send section collects outbound workflows; the adapter catalog stays canonical for provider details.
