---
title: Chat SDK and AI SDK email tools
description: Add one bound, approval-gated sendEmail tool through the optional AI SDK subpath.
---

Install the optional `ai` peer only in applications that import `@opencoredev/email-sdk/ai`.

```bash
bun add ai @opencoredev/email-sdk
```

## Create the tool

```ts title="src/agent/email-tools.ts"
import { createEmailTools } from "@opencoredev/email-sdk/ai";
import { email } from "../../email.js";

export const emailTools = createEmailTools({
  client: email,
  from: "Acme <hello@acme.com>",
});
```

The helper returns `tools: { sendEmail }` and `toolApproval: { sendEmail: "user-approval" }`.

## Compose with AI SDK 7

```ts title="src/agent/respond.ts"
import { streamText } from "ai";
import { emailTools } from "./email-tools.js";

const result = streamText({
  model,
  tools: emailTools.tools,
  toolApproval: emailTools.toolApproval,
  experimental_toolApprovalSecret: process.env.TOOL_APPROVAL_SECRET,
});
```

AI SDK 6 reads the tool-level `needsApproval: true` fallback. AI SDK 7 applications should pass the returned top-level `toolApproval` map.

## Model-visible input

The strict schema accepts only `to`, `subject`, `text?`, and `html?`, with at least one body. The application binds `from`. The model cannot select adapters, fallback, retry, headers, metadata, idempotency, schedule, attachments, CC, or BCC.

Approved execution forwards the abort signal, derives `email-tool:${toolCallId}` as the idempotency key, and returns only `{ status: "sent", adapter, id? }`.

The tool converts send failures into `Error("Email could not be sent.")` so provider bodies and raw errors do not reach the model.

## Approval UI

Render the exact recipients, subject, and body before approval. A denied request is terminal and should not be retried by prompt or application code.

`@opencoredev/email-sdk/agent-tools` remains importable for v1 but is deprecated and receives only critical fixes.
