Keep email templates in `ui/email`, render them on the server, and send the result through your normal Email SDK client.

## Install

```bash
bun add react react-dom @react-email/render
```

## Create a template

```tsx title="ui/email/welcome.tsx"
import {
  EmailButton,
  EmailCard,
  EmailHeading,
  EmailText,
  ShadcnEmail,
} from "@opencoredev/email-sdk/react";

export function WelcomeEmail({ name }: { name: string }) {
  return (
    <ShadcnEmail preview="Your account is ready.">
      <EmailCard>
        <EmailHeading>Welcome, {name}</EmailHeading>
        <EmailText>Your account is ready.</EmailText>
        <EmailButton href="https://acme.com/dashboard">
          Open dashboard
        </EmailButton>
      </EmailCard>
    </ShadcnEmail>
  );
}
```

## Render and send

```tsx title="src/send-welcome.tsx"
import { renderEmail } from "@opencoredev/email-sdk/react";

import { email } from "./email";
import { WelcomeEmail } from "../ui/email/welcome";

const content = await renderEmail(<WelcomeEmail name="Ada" />);

await email.send({
  from: "Acme <hello@acme.com>",
  to: "ada@example.com",
  subject: "Welcome to Acme",
  ...content,
});
```

`renderEmail` returns `html` and `text`. Routing, retries, fallback, scheduling, hooks, and telemetry still go through the normal send pipeline.

## Use your shadcn theme

The included components use email-safe inline styles with shadcn-style semantic colors. Start with the light or dark theme, then override the resolved values you use in your app.

```tsx
<ShadcnEmail
  theme={{
    primary: "#7c3aed",
    primaryForeground: "#ffffff",
    radius: 10,
  }}
>
  {/* email content */}
</ShadcnEmail>
```

Use these email components instead of importing your app's normal shadcn components. Browser components can rely on Radix, Tailwind, CSS variables, and client JavaScript that inboxes do not support.

- [See every component](/docs/ui/react-email/components)
- [Copy a complete block](/docs/ui/react-email/blocks)

The React entry point is server-only and optional. Apps that never import `@opencoredev/email-sdk/react` do not need its React peer dependencies.
