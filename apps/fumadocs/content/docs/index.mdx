Send transactional email through one typed client. Pick an adapter, create the client, and call `send`.

<Card title="Send your first email" href="/docs/getting-started/quickstart" description="Install one adapter, validate the message, and send it from Node.js or Bun." />

## Start here

```ts title="email.ts"
import { createEmailClient } from "@opencoredev/email-sdk";
import { resend } from "@opencoredev/email-sdk/resend";

export const email = createEmailClient({
  adapters: [resend({ apiKey: process.env.RESEND_API_KEY! })],
});
```

Now call `email.send({ from, to, subject, text })`. Add retries, fallback, plugins, and more adapters only when you need them.

## What ships

- 22 provider API adapters plus SMTP, 23 adapters total
- Literal route-name inference across `send`, `validate`, `adapter`, and `withAdapter`
- Sequential retries and delivery-aware fallback routes
- Independent `sendMany` calls and personalized `sendPersonalized` delivery
- Hooks, middleware, built-in plugins, and no-network test adapters
- Provider-native scheduled sends with `sendAt`
- A Node-shebang CLI for discovery, setup checks, and validated dry runs
- Optional AI SDK tools and a separate stdio MCP server for agent workflows

## Runtime boundary

Email SDK is ESM-only and server-side. It supports Node.js 20+ and Bun 1.1+. Keep API keys and SMTP credentials out of browser bundles.
